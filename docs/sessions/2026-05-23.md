## What shipped

### Bug fix ‚Äî data-loss root cause eliminated

**Problem:** `AKTENRAUM_DATA_DIR` was unset in `docker/.env`, causing Docker Compose to fall back to `${HOME}/aktenraum`. On Windows, `HOME` resolves differently depending on the shell that invokes `docker compose` (Git Bash ‚Üí `/c/Users/Admin`, PowerShell ‚Üí different, Task runner ‚Üí different again). Each fresh start could write postgres data to a new path, initialise an empty database there, and silently abandon the previous data. The data was never deleted ‚Äî just stranded at the previous path. This caused two data-loss incidents (May 20, May 23).

**Fix:**
- `docker/.env` ‚Äî `AKTENRAUM_DATA_DIR=D:/aktenraum` hardcoded (pinned to Windows D: drive; survives any shell, any Docker Desktop reset scenario)
- `docker/.env.example` ‚Äî `AKTENRAUM_DATA_DIR=` uncommented with an expanded warning comment explaining the Windows-specific risk

### Bug fix ‚Äî auto-approve silently disabled after env rebuild

`AUTO_APPROVE_CONFIDENCE` and `AUTO_APPROVE_TYPES` were missing from `docker/auto-tagger.env` after `bootstrap-secrets.sh` rebuilt it from the template (the example file had different defaults). Added back:

```
AUTO_APPROVE_CONFIDENCE=0.90
AUTO_APPROVE_TYPES=Rechnung,Kontoauszug,Gehaltsabrechnung,Lohnsteuerbescheinigung
LOW_CONFIDENCE_THRESHOLD=0.70
USE_CORRESPONDENT_HISTORY=true
QDRANT_URL=http://qdrant:6333
```

Verified via logs: `routing_decision reason=auto_approved` firing on Rechnung at 0.95 confidence.

### Bug fix ‚Äî backup repo never initialised

The restic backup repository at `D:/aktenraum/backup/restic-repo` was empty ‚Äî `entrypoint.sh` runs `restic backup` but never `restic init`. Initialised manually; first snapshot taken. The `task setup` flow now includes a `restic init` step (idempotent: exits cleanly if repo already exists).

### Operational UX ‚Äî simple commands + recovery script

**`scripts/fix-token.sh`** (new) ‚Äî complete token-rotation script:
1. Waits for Paperless to be ready (retries for 60 s)
2. Mints a new token using `PAPERLESS_ADMIN_PASSWORD` from `docker/.env`
3. Writes the token into both `auto-tagger.env` and `aktenraum-api.env` via safe temp-file pattern (no `sed -i` risk)
4. Recreates both services

**`Taskfile.yml`** ‚Äî new tasks at the top:

| Task | What it does |
|---|---|
| `task start` | `docker compose up -d` with friendly output |
| `task stop` | `docker compose down` with friendly output |
| `task status` | `docker compose ps` |
| `task recover` | calls `scripts/fix-token.sh` ‚Äî fixes 401 errors in ~30 s |
| `task destroy` | stops stack + deletes `AKTENRAUM_DATA_DIR` (requires typing `DELETE`) |
| `task setup` | complete first-time setup: secrets ‚Üí stack ‚Üí token ‚Üí paperless bootstrap ‚Üí backup init ‚Üí first snapshot |
