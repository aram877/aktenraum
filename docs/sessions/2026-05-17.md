## What shipped

User-reported failure mode: `/ask` returns "Ich konnte das in den Dokumenten nicht finden." in the prose while the SPA still renders the retrieved candidate documents as source cards underneath. Looks like the AI is lying about its own search.

Root cause traced to `services/aktenraum-api/src/aktenraum_api/ai/router.py`:

- Both the JSON `/answer` (line ~200) and the streaming `/answer/stream` (line ~497) branches contain a "if the model wrote prose but emitted zero `[Quelle: N]` markers, back-fill citations from the retrieval set" rule. Intent: prose without a verifiable source is useless. Side-effect: when the prose IS the prompt-baked denial template ("Ich konnte das in den Dokumenten nicht finden."), the back-fill triggers and the SPA shows source cards beneath a "not found" message.

Fix:

- New `_DENIAL_RE` regex + `_is_denial_answer(text)` helper next to the existing `_is_degenerate_answer`. Detects the prompt-baked template plus four common small-model variants ("nicht in den Dokumenten", "keine passenden Dokumente", "keines der Dokumente enth√§lt/nennt/beantwortet"). Length-capped at 200 chars so partial answers like "Du hast bei X gebucht. Die Preise konnte ich nicht finden, aber die Buchungen sind verlinkt." are NOT treated as denials ‚Äî they carry real content and keep their citations.
- Both back-fill branches now gate on `not _is_denial_answer(answer_text)`. Denial prose ‚Üí `citations = []`. Real prose ‚Üí back-fill as before.

Tests added in `services/aktenraum-api/tests/test_ai_answer.py`:

- Five unit tests for `_is_denial_answer` (template hit, common variants, real-prose negatives, empty input, long-partial negative).
- One e2e test for JSON `/answer` denial path.
- One e2e test for streaming `/answer/stream` denial path.

Total: 519 passed (was 512), ruff clean.

CLAUDE.md updated with one row in the `/ask` description noting the new behaviour so future sessions don't reintroduce the back-fill.

## Third pass: web-trash-management (OpenSpec change)

Discovered while debugging the user's "delete also from Paperless trash" request: the existing `paperless_gw.delete_document` had a docstring claiming hard-delete ("Paperless does not soft-delete‚Ä¶") that was false. Paperless 2.x always soft-deletes ‚Äî every doc the user ever clicked L√∂schen on has been sitting in `/api/trash/` recoverable for 30 days, then auto-purged. The SPA never surfaced any of that. Same investigation surfaced an orphaned-Qdrant-chunks bug ‚Äî soft-delete never touched the vector store, so RAG could still answer from "deleted" docs.

OpenSpec change: `openspec/changes/web-trash-management/` (proposal, design, two delta specs, tasks). Design picked the two-step Trash ‚Üí empty model over one-shot hard-delete (per user) and bundled the Qdrant cleanup with the hard-delete path.

**What landed:**

1. **Three new gateway methods** in `paperless_gw.py`: `list_trashed_documents(page, page_size, ordering)`, `restore_documents(doc_ids)`, `empty_trash(doc_ids|None)`. Plus the corrected `delete_document` docstring explaining that Paperless soft-deletes and pointing to `empty_trash` as the hard-delete path.

2. **New `aktenraum_api/trash/` package** (schemas / service / router) mirroring `inbox/`. Service exposes `list_trashed`, `restore`, `delete_forever`, `empty`; hard-delete paths additionally call `vector_store.delete_by_doc_id(doc_id)` after Paperless success. Qdrant cleanup is best-effort ‚Äî logs `trash_qdrant_purge_failed` but never fails the user request. New endpoints: `GET /api/trash/`, `POST /api/trash/{id}/restore`, `POST /api/trash/{id}/delete`, `POST /api/trash/empty`. All auth-gated; ordering allowlisted; 404/502/409 error mapping mirrored from `inbox/`.

3. **New `get_vector_store_optional` dependency** in `ai/deps.py` returns `request.app.state.rag_vector_store` (or None when `QDRANT_URL` is empty). Trash service handles None as a no-op so RAG-off installs stay green.

5. **Trash query lib** (`apps/web/src/lib/trash.ts`) with `useTrashList`, `useTrashCount` (30s staleTime), `useRestoreFromTrash`, `useDeleteForever`, `useEmptyTrash`, plus a `trashDaysRemaining` helper.
