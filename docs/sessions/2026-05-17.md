## What shipped

User-reported failure mode: `/ask` returns "Ich konnte das in den Dokumenten nicht finden." in the prose while the SPA still renders the retrieved candidate documents as source cards underneath. Looks like the AI is lying about its own search.

Root cause traced to `services/aktenraum-api/src/aktenraum_api/ai/router.py`:

- Both the JSON `/answer` (line ~200) and the streaming `/answer/stream` (line ~497) branches contain a "if the model wrote prose but emitted zero `[Quelle: N]` markers, back-fill citations from the retrieval set" rule. Intent: prose without a verifiable source is useless. Side-effect: when the prose IS the prompt-baked denial template ("Ich konnte das in den Dokumenten nicht finden."), the back-fill triggers and the SPA shows source cards beneath a "not found" message.

Fix:

- New `_DENIAL_RE` regex + `_is_denial_answer(text)` helper next to the existing `_is_degenerate_answer`. Detects the prompt-baked template plus four common small-model variants ("nicht in den Dokumenten", "keine passenden Dokumente", "keines der Dokumente enthält/nennt/beantwortet"). Length-capped at 200 chars so partial answers like "Du hast bei X gebucht. Die Preise konnte ich nicht finden, aber die Buchungen sind verlinkt." are NOT treated as denials — they carry real content and keep their citations.
- Both back-fill branches now gate on `not _is_denial_answer(answer_text)`. Denial prose → `citations = []`. Real prose → back-fill as before.

Tests added in `services/aktenraum-api/tests/test_ai_answer.py`:

- Five unit tests for `_is_denial_answer` (template hit, common variants, real-prose negatives, empty input, long-partial negative).
- One e2e test for JSON `/answer` denial path.
- One e2e test for streaming `/answer/stream` denial path.

Total: 519 passed (was 512), ruff clean.

CLAUDE.md updated with one row in the `/ask` description noting the new behaviour so future sessions don't reintroduce the back-fill.

## Second pass: code-review-cleanups (OpenSpec change)

Triaged a 7-point external code review and shipped the five items judged worth doing for a localhost-only baseline. Two items deliberately skipped (rate limiting — only matters when exposing beyond localhost; monorepo ceiling — flagged as a future-only concern by the reviewer).

OpenSpec change: `openspec/changes/code-review-cleanups/` (proposal, design, two delta specs, tasks).

**What landed:**

1. **Approve fires propagation immediately** (review item #2). New auto-tagger webhook `POST /trigger/propagate` mirrors the existing `/trigger/extract` (same `WEBHOOK_SECRET` auth, same 202/400/401/503 responses, shared `_check_secret` / `_parse_doc_id` / `_enqueue_or_503` helpers). Propagation now runs queue-based (`asyncio.Queue[int]` `propagation_queue`) — both the new webhook and the existing 30s safety-net poller enqueue onto it. The worker re-verifies `ai-approved` on dequeue to defend against the webhook/poller race. aktenraum-api's `POST /api/inbox/{id}/approve` fires `ping_auto_tagger(trigger="propagate", timeout=2.0)` after the lifecycle-tag swap; failure is swallowed (poller is the safety net). Approve-to-propagated latency drops from up to 30s to <1s on the happy path.

2. **HuggingFace cache survives rebuilds** (review item #6, scoped down). Lifespan warm-up already exists; the gap was that every `docker compose up -d --build aktenraum-api` blew away the 600 MB `bge-reranker-v2-m3` cache. Added named volume `aktenraum-hf-cache` at `/home/appuser/.cache/huggingface`, pinned `HF_HOME` and `HUGGINGFACE_HUB_CACHE` env vars so the location is explicit regardless of future Dockerfile USER changes. CLAUDE.md "5-minute first /ask" gotcha retired; `desktop-app.md` Phase 0.3 corrected (Ollama can't pull this model — it's sentence-transformers, not Ollama-served).

3. **Image pins (review item #4)** — `paperless` → `2.20.15@sha256:6c86cad8…`, `gotenberg` → `8.31.0@sha256:f0d86e8a…`, `tika` → `latest@sha256:2a565f1e…` (digest pin only since the tag/version mapping is ambiguous). All three were on floating tags previously; pinned to the digests of what is currently running so no behaviour change. Bumps become deliberate from here.

4. **ADR-004: two Python services (review item #1)** — captures the deliberate process-isolation rationale (5 long-lived asyncio tasks + sentence-transformers RSS in auto-tagger vs the FastAPI request loop in aktenraum-api), the costs (env duplication, in-cluster HTTP hop), and four "revisit when" triggers (third Python service, single-host RSS pressure, shared-model handle, latency-load-bearing trigger calls). CLAUDE.md Stack section now links to it.

5. **Confidence-vs-correctness eval (review item #3)** — `scripts/eval-confidence-correlation.py` (stdlib-only, host-runnable) joins `ai_confidence` against an "approved-unedited" proxy (correspondent + doctype match) over the propagated corpus. First run on the live stack (n=19 docs): mean confidence **0.978**, all 17 confidence-bearing rows in the "high" bucket, Pearson ≈ -0.035. Predictor has effectively zero variance — confidence-based routing cannot be validated until the corpus diversifies. Decision criterion baked into CLAUDE.md: re-evaluate auto-approve once N≥50 docs span all three buckets; if Pearson <~0.3, retire `AUTO_APPROVE_CONFIDENCE` in favour of rule-based gates.
