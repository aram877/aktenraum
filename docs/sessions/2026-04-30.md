## What we built this session, by feature

### 1. Foundation (already there at session start)
- `aktenraum-api` FastAPI service with JWT-in-httpOnly-cookie auth, Alembic migrations, dual Postgres DB (`paperless` + `aktenraum`).
- React 19 + Vite + Tailwind v4 + TanStack Router/Query SPA at `apps/web/`.
- nginx edge serving `/` (SPA) and proxying `/api/*` to `aktenraum-api:8002`.
- Token never reaches the browser.

### 2. Find Docs — `/find` (`/api/ai/find`)
- Closed-enum `SearchFilter` (20-value `DocumentType`, correspondent, date range, amount range, free text).
- LLM emits the filter; gateway translates to Paperless query (`document_type__id` / `correspondent__id` — the bare names are silently ignored upstream, that gotcha is documented).
- Editable filter chips re-run the search without re-invoking the LLM.
- Each result has Open + Download buttons (click → `DocumentPreviewModal`).

### 3. Ask AI — `/ask` (`/api/ai/answer`)
- Two-step pipeline: filter LLM → retrieval → answer LLM with the top matches' AI metadata as context → German prose answer with citations.
- Hallucinated citation ids dropped server-side (intersected with searched docs).
- Retrieval broadens (drops `text` filter) when any structural field is set, so verbs like "verlängern" don't kill recall.
- **Dual-model config**: `OLLAMA_MODEL` (fast 8B for filter extraction) + `OLLAMA_ANSWER_MODEL` (smart 26B for prose). Same split for Anthropic via `ANTHROPIC_ANSWER_MODEL`.
- nginx `proxy_read_timeout` bumped to 300s for slow local-LLM calls.
- Live verified: *"Wann muss ich meinen Personalausweis verlängern?"* → "Dein Personalausweis läuft am 27.02.2034 ab." with cited doc.

### 4. Inbox review queue — `/inbox` + `/inbox/$id` (`/api/inbox/*`)
- Two-pane review: PDF iframe + editable form for 12 ai_* fields.
- Approve (optional patch + lifecycle swap) / Reject / auto-advance to next pending doc.
- Keyboard shortcuts: `a` Approve, `r` Reject, `j`/`k` next/prev, `Esc` back to list.
- Low-confidence rows highlighted; nav badge with pending count.
- Server-side normalisation (German dates → ISO, monetary → `<ISO><amount>`, string trunc) — user can't trip Paperless validation.
- **Custom-fields PATCH gotcha fixed**: Paperless replaces the full array, so the gateway reads + merges by field id before writing (otherwise patching one field wipes the other 11). Pinned by 4 regression tests.

### 5. Library — `/library` + `/library/$id` (`/api/library/`, `/api/documents/{id}/{detail,fields}`)
- List all non-pending docs with filters (doc type, correspondent, date range, amount range, free text). URL-state filters (bookmarkable). 400ms debounce on form changes.
- Lifecycle badges per row.
- **Row click → `/library/$id` two-pane review**. PDF iframe + editable form for the 12 ai_* fields. Save / Reset / Erneut verarbeiten / Download / Back. Form re-hydrates after save with normalised values.
- Backed by `GET /api/documents/{id}/detail` and `PATCH /api/documents/{id}/fields` — both reuse `inbox.service` so they work on any doc.

### 6. Upload + Reprocess (`/upload`, `/api/documents/upload`, `/api/documents/{id}/reprocess`)
- Drag-and-drop dropzone, single + multi-file, per-file progress bar.
- **Live phase machine** per file: `Bereit → Wird hochgeladen → Paperless verarbeitet → KI klassifiziert → ✓ Inbox / ✓ Bibliothek / ✗ Fehler`. Pollers at 1.5s for the task stage and 3s for the doc-status stage; cancelled cleanly on unmount; 120s ceiling.
- Reprocess: clears all 7 lifecycle tags + best-effort pings the auto-tagger webhook; falls back to the 30s poller. Optional `WEBHOOK_SECRET`.
- "Erneut verarbeiten" button on `DocumentPreviewModal` (Library / Find / Ask) and the `/library/$id` page.

### 7. Processing-status visibility (`/api/documents/in-flight`, `/task/{uuid}`, `/{id}/status`)
- `DocumentSummary.lifecycle_tags` carries state through every endpoint.
- Shared `ProcessingBadge` component: "Wartet auf KI" (no tag) · "In Inbox" (ai-pending) · "Wird übertragen" (ai-approved) · "Verarbeitet" (ai-propagated) · "Niedrige Konfidenz" · "Abgelehnt" · "Fehler".
- Renders on Library rows, Find result cards, Ask citations, Library detail header, modal previews.
- Nav: global "N in Bearbeitung" pill (in-flight count minus inbox count, refetched every 30s, with a pulsing dot).

### 8. Document open + download proxies (`/api/documents/{id}/{preview,download}`)
- Streaming proxy through `aktenraum-api`. `Content-Disposition` forwarded on download so the original filename survives. `Cache-Control: private, max-age=300` on preview.

### 9. `ai_summary_de` longtext migration
- Was a Paperless `string` field with a hard 128-char DB cap, so summaries were always clipped to one sentence + `…`. Migrated to `longtext`.
- New `LONGTEXT_FIELDS` set + `truncate_for_field(name, value)` helper in `aktenraum-core` — both writers route through it, longtext fields skip truncation.
- Bootstrap creates `ai_summary_de` as `longtext` on fresh installs.
- One-shot `scripts/migrate-ai-summary-to-longtext.sh` for existing installs (drops + recreates the field; old clipped values are lost, use Reprocess to backfill).
- **Live result**: doc 8's new summary is 247 chars, 4 sentences, no ellipsis.

## Things to pick up next session

**Loose ends from today**:
- Old summaries on already-classified docs are gone (migration drops them). Optional follow-up: write `scripts/reprocess-all-propagated.sh` to bulk-backfill.
- Library row's correspondent/doc-type still prefers native FK over AI field. So if a user edits `ai_correspondent` in `/library/$id` the row in `/library` shows the old name until they hit Reprocess. One-line change in `library.service._project` if you want the AI field to win.

**Roadmap candidates**:
- **Phase 5 remainder**: saved queries (persist a `SearchFilter` with a name; appears on home/dashboard) + an Errors page (`ai-error` / `ai-propagation-error` recovery view).
- **Phase 7**: taxonomy management (manage tags / correspondents / document types from the SPA — currently you have to use Paperless's admin UI for this).
- **Phase 6**: real RAG via embeddings only if structured-filter Q&A hits a ceiling. So far it hasn't.

**Smaller polish ideas left on the table**:
- Bulk reprocess (cli / button in the Errors page).
- Multi-select for doc type in Library filters.
- Sort-by-amount / sort-by-title column headers in Library.
- Mobile sidebar drawer (Library is desktop-only-friendly).
- Upload metadata (correspondent / doc type) at upload time as a "preset".

**Known gotchas documented in CLAUDE.md** (in case you onboard a collaborator):
- Paperless `?name=` / `?correspondent=` / `?document_type=` filters are silently ignored — use the `__id` / `__iexact` variants.
- `custom_fields` PATCH is full-array replace, not partial upsert.
- Custom-field data type can't be changed in place — drop + recreate.
- Local 8B LLMs are reliable for filter extraction but too small for prose Q&A; pair them with a 14B+ for the answer step.
