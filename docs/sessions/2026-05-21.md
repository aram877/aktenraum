### ProcessingBadge — email + duplicate auxiliary pills

`apps/web/src/components/ProcessingBadge.tsx`

The badge now renders up to three pills side-by-side when auxiliary tags are present:

- **E-Mail** (sky blue) — shown when `email-ingested` tag is present; title "Per E-Mail eingegangen."

### Dismiss-duplicate action (API + SPA)

`services/aktenraum-api/src/aktenraum_api/documents/router.py`  
`apps/web/src/lib/documents.ts`  
`apps/web/src/components/DocumentPreviewModal.tsx`

New endpoint `POST /api/documents/{id}/dismiss-duplicate` removes the `ai-duplicate` auxiliary tag via `gateway.swap_lifecycle_tag`. Returns `{doc_id}`. 404/409/502 mapped per standard gateway-error pattern.

`DocumentPreviewModal` renders an amber info banner with a "Markierung entfernen" button when the doc carries `ai-duplicate`. On success it shows a green confirmation and the `Duplikat?` pill disappears from the Library row (via cache invalidation of `library` + `document-detail` queries).

Note: v1 limitation — the auto-tagger will re-flag the pair on the next propagation in that correspondent. Dismissal is sticky only until then (documented in CLAUDE.md known-gotchas).
