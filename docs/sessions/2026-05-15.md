### Taskfile.yml + Vite dev fixes (commit `374a751`)

A `Taskfile.yml` at the repo root ([Taskfile.dev](https://taskfile.dev) — `brew install go-task`) wraps every common workflow as a discoverable one-liner. 34 tasks. Headline shortcuts: `task bootstrap`, `task up/down/ps`, `task web:dev`, `task api:rebuild`, `task tagger:rebuild`, `task recreate SVC=…`, `task logs SVC=…`, `task test`, `task lint`, `task reprocess ID=…`, `task rag:backfill / rag:eval`, `task backup:run / backup:snapshots`.

While wiring `task web:dev`, fixed two preexisting Vite bugs that prevented hot-reload from working at all against this stack:

- The dev proxy target was `http://localhost:80`, but nginx is on `:8080` (and overridable via `AKTENRAUM_WEB_PORT`). Default to `:8080` and honour `VITE_API_PROXY_TARGET`.
- Vite 5+ rejects unknown `Host` headers, so LAN-device access would 403. Bind to `0.0.0.0` and set `allowedHosts: true` so a second device on the LAN can hit `http://<dev-machine-ip>:5173` with full hot-reload.

Two new Vite env knobs: `VITE_API_PROXY_TARGET`, `VITE_HOST`. Documented in `docs/configuration.md`.

## Diagnosed but not fixed

The user reported the `ai_title` field showing empty everywhere. Investigation:

- The previous session's fix (commit `09ef7ca` — Python-side synthesis when the LLM drops `ai_title`) is in the running auto-tagger container; live re-extraction of doc 27 confirmed it works (`ai_title = "Ausweis Karte Mahmoud Alosta BKK Gilde"` after reprocess).
- The issue is that every doc in the corpus was last modified on or before May 4 (the latest before the fix). The fix only runs at extraction time, so old docs have no field 15 (`ai_title`) at all.
- The correct cheap fix is a backfill script that synthesises `ai_title` from existing `ai_document_type` + `ai_correspondent` + `ai_issue_date` for every doc missing field 15 — no LLM call, runs in seconds for 24 docs.
- Pending user confirmation before running.
