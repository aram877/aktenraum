## What shipped

### Bug fixes

**`task backup:check` / `task backup:verify` / `task backup:snapshots` failed silently**
(`Taskfile.yml` lines 361, 366, 371)

`RESTIC_REPOSITORY` is set to `/repo` inside `entrypoint.sh`, not as a
container environment variable. Tasks that call `restic` directly (bypassing
`entrypoint.sh`) had no repo path and exited with
"Fatal: Please specify repository location". Fixed by adding
`-e RESTIC_REPOSITORY=/repo` to all three task `exec` calls.

**Phase 0.4 auto-init guard was untestable**
(`docker/backup/entrypoint.sh` line 8)

The script hardcoded `export RESTIC_REPOSITORY=/repo`, which overrode the
`-e RESTIC_REPOSITORY=/tmp/does-not-exist` the Phase 0.4 test passes via
`docker exec`. The guard fired, but against the real `/repo` (which exists),
so it always succeeded — the test could never exercise the "missing repo"
branch. Changed to `: "${RESTIC_REPOSITORY:=/repo}"` so the env var is
respected when supplied and defaults to `/repo` when absent.
